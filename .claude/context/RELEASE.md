# RELEASE.md

This file documents release procedures. See the main copy at [../../doc/RELEASE.md](../../doc/RELEASE.md).

## Release Checklist

- [ ] Version bump (semantic versioning)
- [ ] Changelog updated
- [ ] Tests pass
- [ ] Linting passes
- [ ] Type checks pass
- [ ] Security scan passes
- [ ] Performance baselines met
- [ ] Documentation complete
- [ ] Build succeeds
- [ ] Migration guide (if breaking change)
- [ ] Release notes prepared

## Semantic Versioning

- MAJOR: Breaking changes
- MINOR: New features (backward compatible)
- PATCH: Bug fixes

## Verification

All verification must pass before release:
- Tests
- Linting
- Type checking
- Security scanning
- Performance expectations
- No security vulnerabilities
